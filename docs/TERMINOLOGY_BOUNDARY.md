# ATCB Terminology Boundary

This document is a review artifact for a fictional-data prototype. It documents neutral terminology for conceptual comparison and does not introduce runtime authority, private data, identity material, corpus exchange, ownership transfer, or system merger.

## Preferred Neutral Terms

- continuity-governance engine
- fictional-data prototype
- deterministic local system
- review artifact
- state recovery
- memory integrity
- conceptual comparison

## Translation Table

| Poetic / conceptual term | Neutral technical term |
|---|---|
| alive | operationally continuous |
| death | continuity failure |
| betrayal | boundary violation or trust rupture |
| soul signature | identity kernel |
| memory wound | unresolved conflict record |

## Usage Rule

Poetic or conceptual language may appear only when explicitly labeled as conceptual language and translated into neutral technical terms. Runtime outputs, generated handoff summaries, and review page headings should use neutral technical language.
